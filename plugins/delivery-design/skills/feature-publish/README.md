# feature-publish

Publishes locally generated ticket files to Linear or GitHub Issues.

## When to use

Use this after `feature-tickets` has generated local `.md` files and you have reviewed them. It is the final step in the feature pipeline:

```
feature-brief → feature-design → feature-plan → feature-tickets → feature-publish
```

Do not run this skill until you are satisfied with the ticket files. Tickets created in Linear or GitHub cannot be bulk-deleted easily — review the local files first.

## Requirements

A connected project management MCP is required. Supported tools:

- **Linear** — via the Linear MCP
- **GitHub Issues** — via the GitHub MCP

## Inputs

| Input | Required | Notes |
|---|---|---|
| `tickets/` folder | Yes | Generated by `feature-tickets`; must contain `_epic.md` and story files |
| Project / team / repo | Yes | Asked at runtime — where to create the tickets |

## Output

Tickets created directly in Linear or GitHub Issues:

- **1 Epic** (Linear) or **1 Milestone** (GitHub) — from `_epic.md`
- **N Stories** — one per story file, each linked to the Epic with phase and type labels applied
- Dependency links (blocks / blocked-by) between stories

## How it works

1. Reads all files in `tickets/`, parsing frontmatter for metadata
2. Detects the connected project management tool
3. Asks which project, team, or repo to target
4. Presents the full deployment plan for explicit confirmation
5. Creates the Epic first, then Stories in phase order
6. Links dependencies between stories once all tickets exist
7. Presents a summary with links to every created ticket

## Usage guidelines

This skill runs entirely within the main agent — no subagents are spawned. The work is primarily API calls to the project management tool.

| Setting | Recommendation |
|---|---|
| Model | **claude-sonnet-4-6** is sufficient. |
| Effort | **Low.** The content is already in the files; this skill maps and uploads it. |

The skill asks for explicit confirmation before creating anything. If the deployment fails partway through (e.g. a network error), re-running the skill may create duplicates — check the destination tool before retrying.

## Pipeline

```
feature-brief  →  feature-design  →  feature-plan  →  feature-tickets  →  feature-publish
idea → PRD        PRD → Tech Spec   Tech Spec → Plan  Plan → local files  files → tickets
```

**This is the final step.** After deployment, pick up individual stories and pass them to an agent with the relevant codebase context to execute.
